## Interpretation

1. **Length is the decisive, robust win for the LoRA.** Raw `gpt-4o-mini` sits ~128–135 chars (earth-mover) away from Bohdan's per-bubble length — it writes long, paragraph-style replies. The LoRA is ~1.5–2.9 away (it matches his terse, multi-bubble texting). The Δ CI excludes 0 in both seeds → not noise.
2. **Punctuation register: LoRA matches the "no `!`" rule exactly** (0.00 vs the API's ~0.63). Under the thin prompt the base model reverts to its peppy assistant register.
3. **Message shape is a statistical tie.** `shape_js` favors the LoRA numerically but the Δ CI spans 0 at the n=300 noise floor (~0.02–0.03) in both seeds — call it a tie, not a LoRA win.
4. **Opener variety / diversity:** LoRA opens more variably (higher entropy) but repeats whole replies slightly more (distinct 0.96–0.99 vs 1.0) — mild, watch for mode-collapse at scale.

## Caveats (do not over-read this)

- **Controlled arm only — this is NOT the shipped product.** Both backends ran on the *thin* prompt. The *deployed* API path adds the rich ~1600-token RAG prompt, retrieved few-shot, register/shape directives, and the `)`/`!` logit-bias — machinery built specifically to fix the exact register gaps measured here. The shipped API would score far better on length and `!`. **This experiment isolates raw model voice (weights), not "which product ships better."** That is **arm A (production-realism)**, still to build (needs a `turn_id`-tagged hold-out + retrieval exclusion — see the design doc §7).
- **LoRA near-copy of training replies (~13–15%) is essentially the natural short-text reuse floor — NOT overfitting (resolved 2026-06-02).** Measured baseline: Bohdan's *own real held-out* replies near-match training replies **11.1%** of the time (short casual texts like `ок`, `та норм`, `+` recur naturally). The LoRA sits at 14.7% — only ~3.6pp above that floor — and near-matches *unseen real held-out* replies just **2.7%** of the time (it generates in-distribution style, it does not parrot specific lines). The API is 0% (it writes novel verbose prose). The provisional ε≤5% guardrail was the wrong reference for a short-text domain; the right reference is the ~11% natural floor, which the LoRA barely exceeds.
- **Automatic metrics are surface proxies.** None of them measure "feels like Bohdan" directly. The **blind human panel is the real verdict** (kit built, below).
- **Single decode per item at temp 0.8.** CIs here are sampling (bootstrap) uncertainty; decode-variance and a greedy stable-anchor run are cheap follow-ups (configurable in the runner).

## The verdict gate — blind human panel (built, awaiting Bohdan)

`reports/main/human_eval/rater.html` — 100 blind, randomized forced-choice pairs ("which reply is more like something you'd send?"). Score `choices.json` against `key.json` → win-rate + Wilson CI. This resolves the actual target the metrics only approximate.

## Provisional recommendation

Under identical minimal prompting, the fine-tuned 3B is **much closer to Bohdan's register** (length, punctuation) than raw `gpt-4o-mini` — exactly what the fine-tune was for — and runs locally at $0/reply with comparable median latency. But the **ship decision** for the bot needs two things this run does not yet provide: (1) **arm A** (the API *with* its production scaffold) and (2) the **blind human win-rate**. (The copy-rate caveat is now resolved — the LoRA's near-copy ≈ the ~11% natural short-text floor, not overfitting.) Treat this as a strong, honest *first* result, not the final verdict.
